📘 Paging — What & Why

Idea: Logical memory divided into fixed-size pages; physical memory into equal-sized frames.

Example: If page size = 4KB and process = 10KB, it occupies 3 pages. Page table maps each page → frame.

📗 Segmentation — What & Why

Idea: Divide memory based on logical sections like Code, Stack, and Data. Each has different length.

It allows easier sharing and protection but may cause external fragmentation.

📙 Heap Allocation — Strategies

First Fit: Allocate first suitable free block.

Best Fit: Allocate smallest possible free block.

Buddy System: Split memory blocks in powers of two for easier merging and splitting.
